Proposal

Roadshow Intelligence Platform

Data-driven event scheduling, dealer ranking, and area intelligence for WeBuyVintage.

Prepared for Liam Vasey, Founder 27 March 2026

1. Executive Summary

WeBuyVintage has scaled from zero to 2,800+ roadshow events per year in under four years — a remarkable achievement built on instinct, hustle, and strong dealer relationships. This proposal outlines a data platform that turns your events, customer, and dealer data into a decision-making engine, so growth from here is guided by evidence rather than gut feel.

The core objective: maximise return on every hour of dealer time in the field.

We propose building five interconnected analytical models, delivered as an internal tool your operations team can use day-to-day, with a one-off build phase followed by an ongoing software subscription.

3. What We'll Build

Model 1: Event Repeat Timing Engine

"When should we return to [town X]?"

  • Analyses historical event performance by location over time (revenue, items purchased, footfall, margin)
  • Identifies the optimal "cooldown" period before an area is ready to be revisited — too soon and the well is dry, too late and competitors move in
  • Diminishing returns analysis — quantifies exactly how yield decays with each repeat visit to the same area. Answers: "Does the 3rd visit to Tunbridge Wells still make money? What about the 5th?" Maps the yield curve per location type so you can see where you're past the point of diminishing returns and should rotate to fresh areas
  • Factors in seasonality (e.g. post-Christmas clear-outs, spring cleaning, probate cycles)
  • Factors in "replenishment rate" — how quickly an area's pool of sellable items regenerates between visits
  • Outputs a rolling calendar recommendation: which locations are "ripe" for a return visit this month
Key inputs: Event date, location, revenue, items bought, category mix, footfall/attendance estimates

Model 2: Dealer Performance Ranking

"Who are our best dealers, and where do they perform best?"

  • Creates a composite dealer score based on: total value acquired per event, margin generated (if resale data available), item volume, customer satisfaction signals, category expertise
  • Controls for location quality — a dealer in a strong area shouldn't get credit for the postcode; equally, a strong dealer in a weak area should be recognised
  • Identifies dealer specialisms (e.g. Dealer A is exceptional at jewellery but average at cameras)
  • Flags underperformance early — if a dealer's numbers are trending down, surface it before it costs real money
Key inputs: Dealer ID per event, items purchased per dealer, amounts paid, category breakdown, any customer feedback data

Model 3: Customer & Area Intelligence

"Where are our best customers, and where haven't we been yet?"

  • Maps all customer postcodes to build a heatmap of demand density and spend quality
  • Segments customers by value tier (one-off sellers vs. repeat high-value customers)
  • Identifies white space — high-potential areas (by demographics, wealth indicators, housing age/type) where you've never run an event
  • Ranks untapped postcodes by predicted yield, so you're not guessing where to expand
  • Cross-references with competitor event locations (where data is available) to find underserved markets
Key inputs: Customer postcodes, transaction values, item categories, event attended, repeat visit flag

Enrichment data we'll layer in: ONS demographic data, house price data, population age profiles, Index of Multiple Deprivation, rural/urban classification

Model 4: Dealer Capacity Planning

"Which regions need more dealers?"

  • Compares current dealer deployment against predicted demand by region
  • Identifies bottlenecks — areas where event frequency is capped by dealer availability, not demand
  • Models the revenue impact of adding a dealer in each region, so you can prioritise recruitment
  • Accounts for dealer travel radius and availability to produce realistic deployment scenarios
Key inputs: Dealer home locations, availability patterns, current event assignments, regional event revenue data

Model 5: "What Makes a Great Roadshow Area?" — Pattern Analysis

"What do our best-performing locations have in common?"

This is a one-off strategic deep-dive delivered as a report alongside the platform, covering:

  • Statistical analysis of your top-performing vs. bottom-performing event locations
  • Identification of the external factors that predict success: population demographics, average age, home ownership rates, house prices, distance to nearest competitor, venue type, day of week, time of year
  • A scoring model that rates any UK postcode for roadshow potential — before you've ever been there
  • Clear, actionable findings: "Your best areas tend to be market towns with 10k–30k population, 55+ age skew, above-average home ownership, and no competing roadshow within 20 miles"
Deliverable: Written report + interactive scoring tool embedded in the platform

4. How It Will Work in Practice

The Platform

A web-based internal dashboard (accessible to your ops team, not public) with:

  • Event Calendar View — recommended schedule with colour-coded priority (green = high predicted yield, amber = moderate, red = too soon / low potential)
  • Dealer Leaderboard — live ranking with drill-down into individual dealer performance by category, region, and trend over time
  • Area Explorer — interactive map showing customer heatmaps, white space opportunities, and postcode-level scoring
  • Capacity Dashboard — regional view of dealer supply vs. demand, with recruitment priority flags
  • Data Upload — simple CSV/Excel upload or direct integration with your existing systems to keep models fresh

How models stay accurate

Models are only as good as the data feeding them. We'll establish:

  • A direct integration with your existing Google Sheets event data — automated ingestion, no manual exports
  • Monthly model refresh cycles so recommendations reflect the latest events
  • Quarterly model performance reviews to tune accuracy
