Efficiency Formulas Quick Reference

The equations behind majaco's time-based measurement framework. For the full methodology, see the Time Hierarchy page.

Time Hierarchy

Every level is formed by subtracting a loss category from the level above. The result is Potential Time — the theoretical minimum time to produce your actual good output at bottleneck speed.

Calendar Time — all time in the period (24 × 7 × weeks)
− Non-working Time →
Shift Time — CT − NWT — factory open, shifts rostered
− Schedule Loss →
Planned Production Time — ST − ScL — scheduled to produce
− Planned Downtime →
Operating Time — PPT − PDT — expected to run
− Unplanned Downtime (UPDT) →
Up Time — OT − UPDT — actually running
− LTSR, LTMW →
Potential Time — Good Output / BNS — theoretical minimum

Core Formulas

Potential Time = Good Output / Bottleneck Speed (BNS)
Run Speed = Total Output / Up Time
LTSR = Good Output / Run Speed − Good Output / BNS
LTMW = Waste Output / Run Speed
UPDT = Actual Operating Time − Up Time
Planning Rate = BNS × Operating Efficiency (planned)
Verification: Actual OT = Potential Time + LTSR + LTMW + UPDT  (must balance)

Efficiency Metrics

MetricFormulaAudienceConstraint
Operating Efficiency Potential Time / Actual OT Shop floor Both
Machine Efficiency / OOE Potential Time / Shift Time Management Both
TEEP Potential Time / Calendar Time Strategic Production-constrained
Output per Labour Hour Good Output / Labour Hours Strategic Sales-constrained
Labour Efficiency Machine Eff × (Potential Crew / Actual Crew) — Sales-constrained

Why this works: Operating Efficiency = Potential Time / Actual OT. Since Potential Time = Good Output / BNS, this simplifies to Good Output / (BNS × Actual OT) — i.e. good output as a proportion of what the line could have produced in the actual operating time. The same logic applies to every metric in the table: the numerator is always the time your good output needed; the denominator is the time window you are measuring against.

Machine Efficiency vs OOE: same formula (Potential Time / Shift Time), different name. “Machine Efficiency” is used in a sales context; “OOE” in a production context. The name signals strategic intent, not a different calculation.

OEE translation: the industry-standard Availability × Performance × Quality is the multiplicative equivalent. majaco’s additive framework (Potential Time / OT) provides the same insight without double-counting speed losses on waste units.

Financial Valuation of Lost Time

Every lost hour has a pound value — but the value depends entirely on your constraint state.

Production-Constrained

Lost Output × Unit Marginal Profit (UMP)

Two UMP variants:

  • With existing labour — higher margin, labour cost excluded (already paid)
  • With new labour — lower margin, labour cost included (overtime or new hires needed)

Recovery calculated at current Machine Efficiency, not 100%.

Sales-Constrained

Labour Hours Saved × Fully Loaded Labour Cost

Additional output has no value — you cannot sell more than the market will buy. The only saving is fewer people or fewer hours for the same output.

The difference matters enormously. The same 2.5 hours of unplanned downtime per day can be worth £5.4M/yr (production-constrained, valued at throughput margin) or £82k/yr (sales-constrained, valued at labour cost saved) — a 66× difference. Getting the constraint diagnosis right is not optional.

Lost time scope:

  • Operational — losses within Operating Time (UPDT, LTSR, LTMW)
  • Tactical — losses within Shift Time (adds Schedule Loss and Planned Downtime)
  • Strategic — losses within Calendar Time (adds Non-working Time)

Diagnostic Quick Reference

LTSR Elevated?

Two possible causes — the fix is different for each:

  1. True slow running — the line deliberately operates below BNS (e.g. operator reduces speed for quality reasons).
  2. LTSS micro-stops — the line runs at full speed between frequent brief stops that are too short to log as UPDT but reduce the average Run Speed.

Diagnose with a lineside time study: observe whether the line speed setting matches BNS, or whether frequent micro-stops are dragging the average down.

UPDT Prioritisation

Use a frequency–duration matrix to prioritise downtime events:

Low DurationHigh Duration
High Frequency Highest value — often missed, frequent short stops that aggregate to large losses Major chronic issue — visible and usually already being addressed
Low Frequency Noise — low total impact Rare catastrophic events — address through preventive maintenance

MECE Verification

Actual OT must equal Potential Time + LTSR + LTMW + UPDT. If it does not balance, a loss category has been mis-allocated. Check Run Speed and BNS inputs first.
