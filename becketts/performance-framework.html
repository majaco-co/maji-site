Data Sources

Unit Conversion Master

SKU-level conversions: packs, cases, pallets, kg — canonical units for every metric
›Pack weight (g nominal) and portions per pack by SKU
›Packs per case, cases per pallet by SKU
›Kg per pack / case / pallet (derived)
›Back/block weight where applicable (pre-slicing unit)
›Empty pack weight (tare) by SKU/pack format — for net weight derivation from checkweigher
›Canonical unit declared per KPI — ensures cross-source consistency

OEE System Data

Run events, downtime events, good/waste output, changeover events
›Run events → Up Time, Operating Time
›Downtime events with reasons → UPDT
›Good + waste output → Run Speed, LTSR, LTMW
›Changeover events → actual C/O durations

Works Order Data

Production schedule, SKU volumes, line assignments
›Daily plan by line and SKU
›Planned volumes (kg / units)
›Changeover schedule → PDT
›Shift Time and Schedule Loss

Hours Data

Labour hours by day, individual, area, direct/indirect
›Total hours per day, tagged to individuals
›Classified by area (line, department)
›Direct vs indirect split
›Overtime hours flagged separately from standard hours
›Skill/role mix — distinguishes skilled vs unskilled and role type on line

Absence / Headcount Data

Planned vs actual headcount, absence type, agency/permanent split
›Planned headcount by line/day from roster
›Actual attendance vs plan — gap = absence
›Absence classified: planned (holiday, training) vs unplanned (sick, no-show)
›Agency vs permanent hours split

Yield Data

Pack weights, portion weights, gripper end, misshapes, non-standard
›Pack weights from checkweighers (giveaway)
›Portion weights from slicers (1–3 portions/pack)
›Gripper end data (trim waste at slicer)
›Misshapes % — product not meeting visual spec
›Non-standard % — downgrades sold at lower £/kg

Raw Material Intake Data

Batch input weights, lot tracking, supplier spec
›Intake batch weight — kg of raw material input to each line/run
›Lot/batch traceability from goods-in to finished pack
›Supplier and raw material spec reference
›Material issued to production per run — stock system / back-flush from ERP

BOM / Recipe Standards

Standard material quantity and cost per SKU — the recipe 'should use'
›Standard material quantity per unit output by SKU (recipe)
›Standard material cost per SKU (priced BOM)
›Material type/grade classification with unit costs
›Yield allowance built into recipe standard

Budget / Forecast Data

Budgeted volumes, standard margins, period targets
›Budgeted production volumes by SKU by period
›Standard margin per SKU from commercial model
›Period forecast (weekly/monthly) for volume and mix
›Overhead absorption rates per line-hour

Planning Standards

Planned direct + indirect man hours for daily volume
›Hours per volume by SKU/line
›Direct + indirect allowances
›Standard planning rates + C/O allowances

Commercial Standards

Direct hours + target £/kg by commercial model
›'Should cost' for direct labour
›Target selling price per kg by SKU
›Benchmark for actual vs allowed
›Raw material £/kg cost — for converting yield losses to £ impact
›Standard material price by material type/grade — for MUV price variance

Plan Attainment (SKU-level)

Did we make what we said we'd make? Actual vs planned output per SKU per day
Plan Attainment % = Actual Good Output ÷ Planned Output (per SKU)
›Over- and under-production flagged
›Pareto: which SKUs consistently miss plan?
›Links to service level and schedule smoothing

Kg / Labour Hour vs Planning Standard (Weekly)

Total productivity — uses direct + indirect hours vs planning standard hours
Actual kg ÷ (Direct + Indirect hrs) vs Planning Std kg/LH
Planning Standards include both direct and indirect allowances — this is the operational planning view
›X-axis: total planning standard hours (volume proxy, mix-adjusted)
›Y-axis: actual kg per total labour hour (D+I)
›Shows: do we get less productive as volume rises?
›Captures all labour including supervision, QA, etc.

Kg / Direct Hour vs Financial Standard (Weekly)

Commercial productivity — uses direct hours only vs financial (commercial) standard hours
Actual kg ÷ Direct hrs only vs Financial Std kg/DH
Financial Standards give direct hours only — this is the commercial / margin view
›X-axis: financial standard direct hours (what the margin assumes)
›Y-axis: actual kg per direct labour hour
›Gap = direct labour cost leakage vs commercial model
›Directly links to margin erosion from labour overspend

£/kg Labour Cost

Total labour cost per kg produced — actual vs commercial standard
£/kg Labour = (Direct hrs × Direct rate + Indirect hrs × Indirect rate) ÷ Actual kg
Decomposes into: agency premium | overtime premium | crewing efficiency | indirect ratio
›Gap to commercial standard = margin erosion from labour
›Split by agency vs permanent cost impact
›Overtime cost separated from base rate overspend
›Weekly trend by line and area

Material Usage Variance (£)

Difference between what materials should have cost for actual output vs what they did cost
MUV = (Std qty for actual output − Actual qty used) × Std price per kg
Decomposes into: yield variance (quantity) | mix variance (grade) | price variance (cost)
›Yield variance: actual material consumed vs recipe standard for good output produced
›Mix variance: using higher/lower grade materials than recipe specifies
›Price variance: actual material cost vs standard price (often procurement-driven)
›Pareto by SKU, line, material type — identifies where material cost leaks

Time Hierarchy Decomposition

Every minute in the factory is classified into exactly one category. The hierarchy below shows how calendar time narrows to productive time, and where each loss category sits.

Calendar Time → Shift Time
Calendar Time = Total hours in period (168 hrs/week)
Shift Time = Calendar Time − non-working time
Machine Efficiency denominator = Shift Time
Shift Time → Actual Operating Time
Schedule Loss = Breaks, no-demand periods
PDT = Changeovers, planned maintenance (at planned duration)
Planning Efficiency = Actual Op. Time ÷ Shift Time
Actual Operating Time → Up Time
UPDT = Stops > threshold + PDT overruns
Up Time = Actual Op. Time − UPDT
Line physically running and producing output
Up Time → Potential Time
LTSR = Time lost below BNS (good output only)
LTMW = Time producing waste at run speed
Operating Eff = Potential Time ÷ Actual Op. Time
Key Equation
Shift Time = Potential Time + LTSR + LTMW + UPDT + PDT + Schedule Loss
Machine Efficiency = Potential Time ÷ Shift Time = Operating Efficiency × Planning Efficiency
