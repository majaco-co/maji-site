Architecture Layers

Each layer has distinct responsibilities, ownership, and SLAs.

1

Source Systems

PLC tag historians, SCADA event logs, MES batch records, ERP production orders. Data stays in native format with minimal transformation at the edge.

2

Integration Layer

Event-driven ingestion via OPC-UA, MQTT, and REST adapters. Schema registry enforces contracts. Dead-letter queues capture malformed payloads for replay.

3

Analytical Model

Dimensional model with conformed dimensions across sites. OEE, yield, speed loss, and downtime facts at 1-second, 1-minute, and shift granularity.

4

Consumption Layer

Pre-aggregated materialized views for dashboards. Real-time streaming endpoints for the maji engine. Row-level security by site, line, and role.

Governance & Quality

Data quality rules enforced at each layer boundary.

Schema Contracts

Every source registers a versioned schema. Breaking changes require a migration plan and downstream impact review before deployment.

Freshness SLAs

Source → Stage: ≤ 5s. Stage → Model: ≤ 60s. Model → Dashboard: ≤ 5 min. Alerts fire on breach.

Lineage Tracking

Column-level lineage from PLC tag to KPI tile. Every transformation is logged, versioned, and reproducible for audit.
